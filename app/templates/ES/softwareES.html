Estructura del servidor

El servidor se ha implementado en un microordenador o microcontrolador, dando la posibilidad de utilizar plataformas como Raspberry Pi, Arduino u otros dispositivos compatibles con el Socket. Proyecto IO. La estructura del servidor se basa en el archivo server. py, que es el enlace central conectado a la base de datos. Del mismo modo, client. py, una aplicación de Flask que también se integra con la base de datos, crea una página cliente dedicada usando Flask y el motor de plantillas. El proceso de generar una página cliente permite al cliente verla compartiendo un servidor local usando la aplicación ngrok.

Seleccionar

Un elemento select está disponible en la interfaz del cliente, que le permite seleccionar comandos pasados desde la base de datos para encapsular y aumentar la seguridad. Después de la autenticación, el usuario pasa comandos al cliente. Para que los comandos se envíen desde el cliente al servidor, es necesario iniciar sesión introduciendo un nombre de usuario y una contraseña. El servidor procesa estos comandos utilizando datos de las lecturas de los sensores y dispositivos de medición conectados. Entonces los resultados se envían de vuelta a server. py, y desde allí a client. py. Como resultado, client. py envía los resultados finales a la página. Esta secuencia de operaciones se basa en Socket. Tecnología IO, que permite la comunicación en tiempo real entre el cliente y el servidor. Además, hay un servidor paralelo a nivel de cliente.
