Server structure

The server has been implemented on a microcomputer or microcontroller, giving the ability to use platforms such as Raspberry Pi, Arduino or other devices compatible with the Socket. IO project. The structure of the server is based on the server. py file, which is the central link connected to the database. Similarly, client. py, a Flask application that also integrates with the database, creates a dedicated client page using Flask and the template engine. The process of generating a client page allows the client to view it by sharing a local server using the ngrok application.

Next

Select

A select element is available on the client interface, allowing you to select commands passed from the database for encapsulation and increased security. After authentication, the user passes commands to the client. In order for commands to be sent from the client to the server, it is necessary to log in by entering a login and password. The server processes these commands using data from the readings of the connected sensors and measuring devices. Then the results are sent back to server. py, and from there to client. py. As a result, client. py sends the final results to the page. This sequence of operations is based on Socket. IO technology, which enables real-time communication between the client and the server. In addition, there is a parallel server at the client level.
